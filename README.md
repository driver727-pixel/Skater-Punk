# Skater-Punk

[![Build test drive package](https://github.com/driver727-pixel/Skater-Punk/actions/workflows/test-drive-pages.yml/badge.svg)](https://github.com/driver727-pixel/Skater-Punk/actions/workflows/test-drive-pages.yml)

Grand Vision  Web/Android based. I am building an 80s/90s cyberpunk delivery game. I have characters on electric skateboards. Where the user builds up to a squad of 6 skateboarders who go on automated routes chosen by the user. The routes run in real time in the background. The game is map, flash card, and garage based.

## Test drive the prototype

The repo includes a zero-build browser prototype in:

- `index.html`
- `styles.css`
- `app.js`

If you do not want to run a local server, use the GitHub Actions workflow artifact instead:

1. Open the repository on GitHub.
2. Go to **Actions**.
3. Open the latest **Build test drive package** run, or push changes to `main` / `cursor/game-initial-files-ea73`.
4. Download the **skater-punk-test-drive** artifact from that run.
5. Extract it and open `test-drive.html`.

Quick links:

- [Open workflow runs](https://github.com/driver727-pixel/Skater-Punk/actions/workflows/test-drive-pages.yml)
- [Open the latest repository runs](https://github.com/driver727-pixel/Skater-Punk/actions)

Included in the artifact:

- `index.html`
- `styles.css`
- `app.js`

This gives you a zero-install test-drive package even if GitHub Pages is still configured for a different source.
